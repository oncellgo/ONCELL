# church-events — 교회일정 일괄 시드 프로토콜

## 배경 — 왜 이 스킬이 있는가

**2026-04-21 사고**: 2026년 연간일정을 시드했는데, 특정 날짜의 이벤트가 달력상 **그 다음 날짜까지 걸쳐 보이는** 현상이 다수 발생. 원인은 타임존:
- 이벤트는 `start_at 00:00 +0800`, `end_at 23:59 +0800`으로 저장됨 (= UTC `16:00` ~ 다음날 `15:59`).
- 브라우저가 **UTC+9 (KST)** 로 설정된 사용자는 `end_at` 이 **로컬 00:59 다음날**로 표시되어 다음 날 칸에 해당 이벤트가 나타남.
- 결과: "유아세례문답이 4/26·4/27 양일에 보인다"는 혼란. 사용자가 DB에 duplicate row가 있는 것으로 오해함.

교정 기록: `lib/events.ts`의 `isAllDayEvent` + `getSGDateKey`를 SG 벽시계 기준으로 고정, `ScheduleView`가 all-day 이벤트는 SG 날짜 하나에만 버킷팅하도록 수정 (2026-04-21 commit).

**그럼에도 시드 단계의 불변 원칙은 지켜야 한다.** 아래를 모든 연간 시드에서 적용.

## 핵심 규칙 (시드 시 반드시)

### 1. 모든 이벤트는 **종일(all-day)** 저장
```
start_at: `${date}T00:00:00+08:00`   (= UTC 전날 16:00)
end_at:   `${date}T23:59:00+08:00`   (= UTC 당일 15:59)
```
- 시간 지정 금지. 교회행사 특정 시간이 적혀 있어도 **종일로 저장**.
- `lib/events.ts`의 `isAllDayEvent()` 가 이 패턴을 인식해 전용 경로로 렌더링.

### 2. 단일 이벤트 (기본)
```js
{ date: '2026-01-04', title: '신년예배' }
```
→ 1 row, rule=null, `start_at`·`end_at` 동일 날짜의 00:00~23:59.

### 3. 기간 이벤트 — **시작 날짜만** 저장할 것과 **다일 확장**을 구분

**시작일만 마커로 남기는 이벤트** (신학적·행정적 "시작" 이벤트):
- `구역모임 시작` — 해당 주 시작일만
- `구역모임 방학` — 방학 시작일만
- `성경대학 개강` — 개강일만
- `수련회 마감` 같은 단일 이정표
- **교회력/절기 시즌**: `사순절 시작`, `대강절 시작`, `고난주간 시작` 등 — **첫날**에만 단일 row (40일 / 4주 펼쳐서 넣지 말 것). 단일 일자 절기(`재의수요일`, `종려주일`, `성금요일`, `부활주일`, `성탄절` 등)도 해당 날짜 단일 row.

→ `days:`·`rule:` 없이 단일 row.

**실제로 연속 여러 날 반복되는 이벤트** (새벽기도회·수련회 본기간):
```js
{ date: '2026-01-05', title: '신년 전교인새벽기도회', days: 5 }   // 1/5~1/9
{ date: '2026-03-30', title: '고난주간새벽기도회', days: 5 }     // 3/30~4/3
{ date: '2026-04-14', title: '전교인 성경통독수련회 1차', days: 4 }
```
→ 1 row + `rule: { freq: 'daily', interval: 1, count: N }`. **N개의 row를 만들지 말 것**. `expandOccurrences`가 on-demand로 펼친다.

**주간 반복** (예: 입교세례교육 4주):
```js
{ date: '2026-03-07', title: '입교세례교육', rule: { freq: 'weekly', interval: 1, count: 4 } }
```

### 4. 읽을 컬럼 / 무시할 컬럼

교회요람 이미지에는 여러 세로 컬럼이 있다. **시드 대상은 아래 세 컬럼으로 한정**:

| 대상 컬럼 | 처리 |
|---|---|
| **교회력/절기** | 날짜 있는 절기 → 시작일 단일 row (§3 마커 규칙) |
| **주일** | 해당 주일에 있는 특별 주일명(예: "추수감사주일", "어린이주일", "온세대 연합예배") |
| **행사** | 일반 행사(당회·제직회·수련회·세미나 등). 기간/반복 구분해 §3 적용 |

**완전히 무시할 컬럼** (다른 시스템/정책):
- `주일예배` — `oncell_worship_services` 테이블 소관. 이미지에 적혀 있어도 `oncell_events`에 절대 넣지 않는다
- `새벽담당` — 교역자 순번. 사용자 일정이 아님
- 계절/학기 구분선·요람 범례·페이지 번호 등 메타 텍스트

### 5. ID 규칙 (중복 삽입 방지)
```
plan{YEAR}-{YYYY-MM-DD}-{NN}-{title-공백제거}
```
- `seed-church-events-2026.mjs`의 기존 패턴 유지.
- 동일 ID upsert → 재실행 안전. 같은 이벤트를 다른 스크립트에서 insert 하지 말 것.

## 실행 프로토콜

이미지/텍스트 입력 받으면:

1. **대상 컬럼만 스캔** — `교회력/절기` · `주일` · `행사` 세 컬럼만. `주일예배`·`새벽담당` 등 나머지 컬럼은 눈으로도 읽지 말고 건너뛴다 (실수로 섞이지 않게).
2. **월별 추출** — 위 세 컬럼에서 `{date, title, days?, rule?}` JSON 배열 생성.
3. **규칙 1·3 적용** — 모두 종일, 기간은 `rule:`/`days:` 단일 row, 시작일 마커는 단일 row.
4. **제외 필터** — §4 "무시할 컬럼"에 해당하는 내용이 실수로 들어갔는지 마지막 검증.
5. **사용자에게 목록 확인** — "다음 N건 시드합니다:" 월별로 보여주고 "ㅇㅇ" 받기.
6. `scripts/seed-church-events-{YEAR}.mjs` 에 `EVENTS` 배열 추가.
7. **dry-run**: `node scripts/seed-church-events-{YEAR}.mjs`
8. 건수 확인 후 `--execute`.

## 화면에서의 동작 보장

모든 호출부는 `isAllDayEvent(start_at, end_at)`이 `true`일 때 **SG 시간(UTC+8) 기준 start_at의 날짜 하나**에만 이벤트를 표시해야 한다. 이미 적용된 곳:
- `components/ScheduleView.tsx` — 월/주 캘린더 버킷팅
- 향후 새 캘린더·목록 화면 추가 시 **반드시 같은 패턴** 적용 (`getSGDateKey(ev.start_at)` 사용).

브라우저 로컬 TZ(`new Date(...).getHours()` 등)를 쓰지 말 것 — KST 사용자에게서 다음날 spillover 재발.

## 부록 A. 주보·구역예배지 게시물 매칭 규칙 (koreanchurch.sg)

주보(`/sunday-worship`)와 구역모임교안(`/cell-teaching`) 화면의 날짜 피커와 `/api/cell-worship`·`/api/sunday-bulletin` 게시물 매칭을 구현·수정할 때 반드시 따를 것.

### A.1 주간 피커 UI (최근 4개 주일)
- 날짜 셀 아래 라벨은 **`Math.ceil(date / 7)`** 기반 `첫째주 / 둘째주 / 셋째주 / 넷째주 / 다섯째주`로 표시.
- **좌우 ‹ › 화살표 없음** — 최근 4개 주일만 스냅샷으로 표시, 과거·미래 주간 네비 불허 (두 페이지 모두 동일).

### A.2 "현재 속한 주의 일요일" 계산
- `mostRecentSunday(today)` 는 오늘이 **일요일이면 오늘**, 그 외면 **다가오는 일요일**을 반환 (= 현재 속한 주의 일요일).
- 즉 월요일부터 이미 그 주의 주일이 "현재 기준 주일"이 된다 (`getDay() !== 0 이면 + (7 - getDay())` 일 이동).
- 과거로 당기는 로직을 **쓰지 말 것**.

### A.3 구역예배지 게시물(`M월 N째 주 구역예배지`) 매칭
- `nthSundayOf(year, month, nth)` 는 **해당 월의 N번째 일요일을 그대로** 반환 (시프트 없음).
- UI에서 날짜 4/19를 "셋째주"로 라벨링 → 매칭되는 게시물은 "**4월 셋째 주 구역예배지**" (동일한 N).
- **과거 구현에서 `-7일 shift`를 적용했던 이력이 있으나 2026-04-22에 제거되었다** — 다시 도입 금지.
- 제목 정규식: `([첫둘셋넷다섯])째\s*주`, 월: `(\d+)월`, 분류: `구역예배지` 포함.

### A.4 주보 게시물(`YYYY년 M월 D일 주보`) 매칭
- `date` 쿼리 = 주일의 YYYY-MM-DD. 게시물 제목에서 `(\d{4})년\s*(\d{1,2})월\s*(\d{1,2})일` 추출해 정확 일치로 매칭.
- 주보의 "미스바 보기" 버튼은 게시물 본문 내 **텍스트나 URL에 `misba`/`미스바`/`mizpah`가 포함된 `<a>`** 중 **가장 마지막 링크**를 그대로 사용. 첫 번째 첨부 이미지 링크를 쓰지 말 것.

### A.5 금지/유의
- 게시물 자동 매칭 규칙을 바꾸기 전 **본 스킬의 A.3/A.4 항목을 업데이트**하고, 리뷰 후 반영.
- 화면에 보이는 "N째주" 라벨과 실제 매칭되는 게시물 이름이 **시프트 때문에 엇갈리는 일이 없도록** 한다 — 사용자 혼란의 가장 큰 원인.

## 부록 B. 대시보드 "N월 교회일정" 월간 목회일정 추출 (미스바 PDF)

대시보드의 "N월 교회일정" 카드는 **`oncell_events` DB가 아니라 미스바 PDF에서 추출**한 월간 목회일정을 보여준다. `/api/monthly-schedule` 엔드포인트가 담당.

### B.1 파이프라인
1. `koreanchurch.sg/noticeandnews` 목록에서 **가장 최근 "주보" 게시물**(`YYYY년 M월 D일 주보`) 선택.
2. 해당 게시물 본문의 `<a>` 태그 중 **href 또는 anchor text에 `misba|미스바|mizpah` 포함**된 링크를 추출 (여러 개면 **가장 마지막 = 게시물 하단**).
3. 해당 URL(미스바 PDF)을 다운로드.
4. `pdf-parse`(v2 class API) 로 전체 텍스트 추출.
5. 텍스트에서 **`M월 목회일정`** 헤더를 regex로 찾고, 그 아래부터 다음 큰 섹션(예: `N월 교회력`, `주간 사역`, `예배 일정`, `Announcement`)까지의 블록 파싱.
6. 라인별로 날짜 포맷(`M/D`, `M월 D일`, `D (요일)`) + 제목 추출 → `{date: 'YYYY-MM-DD', label: 'M/D', title}` 목록 생성.
7. `month` 일치 항목만, 중복(`D|title`) 제거, 날짜 오름차순 정렬해 반환.

### B.2 캐싱 전략 (**반드시 양쪽 적용**)
- **메모리 캐시** (warm invocation용): `Map<cacheKey, Cached>`, TTL 1시간. 같은 Lambda 인스턴스 재호출 시 PDF 파싱 생략.
- **영구 캐시** (Supabase `oncell_app_kv`, key=`monthly_schedule_YYYY_MM`): cold start 에도 유지. 파싱 성공(`items.length > 0`)시에만 쓰기 — 빈 결과를 고정시키지 않음.
- 읽기 순서: memory → KV → upstream. 응답에 `cached: 'memory'|'kv'` 힌트 포함.
- `koreanchurch.sg`·미스바 PDF 다운로드는 비용 있는 외부 호출이므로 **항상 캐시 우선**.

### B.3 UI 규약 (대시보드)
- 카드 헤더: `📅 {month}월 교회일정` · "전체 →" 링크는 `/schedule`.
- 리스트 항목: `[M/D] [제목]` 포맷, 날짜 라벨은 라임 계열(`#065F46`).
- 하단 주의문: `※ 교회의 사정에 따라 일정은 변경될 수 있습니다. (출처: 미스바 목회일정)`.
- 실패/빈 결과는 **에러로 표시하지 않고** "이번달 목회일정 정보가 아직 준비되지 않았습니다." 메시지.

### B.4 파싱 패턴 보강 규칙
- 날짜 없는 일반 문구·페이지 번호(`-- N of N --`)·섹션 헤더는 걸러낸다.
- `M/D(요일) 제목` 형태를 가장 선호, 이어서 `M월 D일 제목`, 마지막으로 월 컨텍스트 기반 `D (요일) 제목`.
- 제목 길이는 2~80자 범위에서만 허용, `M === month` 일치 항목만 포함.

### B.5 새 규칙 추가 시
파싱 실패나 이상 데이터 제보가 오면:
1. `pdf-parse` 결과를 먼저 로그로 찍어 원인 확인 (`console.log(pdfText.slice(0, 500))`).
2. B.1 파이프라인의 어느 단계에서 실패했는지 특정 후, 규칙을 본 스킬에 **먼저 추가**한 뒤 코드 변경.
3. 캐시 무효화는 `kvDelete(monthly_schedule_YYYY_MM)` 또는 `cache` TTL 만료 기다림.
